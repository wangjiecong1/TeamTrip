# TeamTrip Agent Instructions

## Mandatory Visual Design Skill

Before making any TeamTrip frontend UI change, read and follow:

- `docs/skills/teamtrip-visual-design/SKILL.md`

This project-level design skill is the source of truth for TeamTrip visual work. It applies to pages, components, modals, forms, cards, buttons, tags, empty states, loading states, and other visible UI.

Do not introduce visual styles that conflict with the TeamTrip design direction. Do not add new primary colors, radius systems, shadow systems, or decorative styles unless the user explicitly approves the design change.

For functional changes, follow the user's latest request and the current codebase behavior. The design skill constrains visual presentation only; it does not replace product requirements or business logic.
